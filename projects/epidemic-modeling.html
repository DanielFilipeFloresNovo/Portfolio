01 · Overview

Modeling dependency

This project explored chemical dependencies as a population phenomenon by adapting the traditional SIR epidemiological model.

Individuals are divided into susceptible, affected and recovered populations, allowing the application to simulate how dependency may evolve through time under different parameters.

The application was implemented in Java and included numerical methods for solving systems of first-order ordinary differential equations, alongside a visual interface for analysing population trends and statistical data.

02 · Main experience

Adapted SIR model

The population moves through three connected states, producing dynamic trends that can be simulated and compared over time.

SIR

S

Susceptible

Individuals who may develop a chemical dependency.

I

Affected

Individuals currently affected by the dependency.

R

Recovered

Individuals who have moved out of the affected population.

Susceptible variation

dS / dt

Describes the rate at which susceptible individuals leave the initial population.

Affected variation

dI / dt

Represents the balance between new cases and recovery.

Recovery variation

dR / dt

Describes how individuals move into the recovered population.

03 · Numerical analysis

From equations to data

Numerical methods transform the differential model into discrete population values that can be calculated and visualized by the application.

Differential system

First-order ODEs

Population changes are represented through a connected system of ordinary differential equations.

Approximation

Numerical resolution

Numerical methods estimate the population values for each moment of the simulation.

Parameters

Scenario analysis

Different parameter values allow multiple dependency scenarios to be compared.

Results

Statistical output

Calculated values are transformed into readable trends, tables and statistical indicators.

04 · Visual analysis

Population trends

The visual interface allows users to compare the susceptible, affected and recovered populations throughout the simulation.
